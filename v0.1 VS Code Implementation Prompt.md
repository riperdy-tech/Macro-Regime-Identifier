# v0.1 VS Code Implementation Prompt: Macro Regime Intelligence Engine

Use this prompt in VS Code, Cursor, Copilot Workspace, or another coding agent to generate the first working version of the standalone macro regime engine.

This prompt assumes two planning documents already exist:

1. **Version 0.1 Technical Spec: Structured Macro Regime Engine**
2. **v0.1 Config Draft: Sources, Dimensions, and Regime Weights**

The implementation should follow those documents closely.

---

# Primary Instruction

Create a standalone local-first Python project named:

```text
macro-regime-engine
```

The project should classify U.S. macroeconomic regimes using structured macroeconomic and market-implied data.

Version 0.1 should use a **FRED-first** data ingestion approach, local storage, YAML configuration, transparent feature scoring, an explainable rule-based regime classifier, CLI commands, JSON output, markdown report output, and unit tests.

Do **not** implement Reddit, GDELT, FinBERT, BERTopic, SEC filings, FOMC text analysis, dashboards, Supabase, GitHub Actions, or cloud deployment in v0.1.

---

# Core Principles

1. All model assumptions must be config-driven.
2. Do not hardcode macro series, dimensions, feature weights, regime weights, or thresholds in Python.
3. Load sources, dimensions, regimes, and model parameters from YAML files.
4. The engine must run locally.
5. Use DuckDB and Parquet for local storage.
6. The engine should still run if optional data is missing.
7. Required missing or stale data should lower confidence rather than crash the whole run, unless no valid dimension can be computed.
8. Outputs must be human-readable and machine-readable.
9. The project should be testable from the beginning.
10. Prioritize correctness, clarity, and extensibility over speed.

```

Implementation notes:

1. Sort each series by date before transforming.
2. For monthly/quarterly data, use observation count shifts when appropriate.
3. For daily market data, approximate one month as 21 trading days and three months as 63 trading days.
4. For weekly data, approximate one month as 4 weeks and three months as 13 weeks.
5. Avoid lookahead bias in rolling calculations. The rolling mean and standard deviation should use data available up to the current observation.
6. Handle insufficient history by returning `NaN` until enough observations exist.

    ...
```

Formula:

```text
confidence =
    0.30 * data_completeness
  + 0.25 * data_freshness
  + 0.25 * signal_agreement
  + 0.20 * regime_decisiveness
```

Implementation guidance:

1. `data_completeness`: average dimension confidence or required-source coverage.
2. `data_freshness`: average freshness score from dimensions/features.
3. `signal_agreement`: v0.1 can approximate as the average absolute contribution of dimensions supporting the top regime.
4. `regime_decisiveness`: top probability minus second probability, rescaled to `0..1`.

Clamp confidence to `[0, 1]`.
