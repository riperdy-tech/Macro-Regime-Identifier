# v0.1 Config Draft: Sources, Dimensions, and Regime Weights

This document defines the first implementation-facing configuration contract for the standalone **Macro Regime Intelligence Engine**.

The goal is to make the model assumptions explicit before implementation begins in VS Code.

This draft should eventually become four configuration files:

```text
config/
  sources.yaml
  dimensions.yaml
  regimes.yaml
  model.yaml
```

Version 0.1 uses a **FRED-first structured data approach**. Direct BLS, BEA, Treasury, SEC, Fed-text, GDELT, and social/news integrations are deferred to later versions.

---

## 1. Configuration Philosophy

All important model assumptions should live in config, not hardcoded Python.

Config should define:

1. which series are ingested
2. which macro dimension each series supports
3. how each series is transformed
4. whether higher values are good, bad, inflationary, restrictive, or risk-positive
5. how each feature is weighted
6. whether each feature is required or optional
7. how stale data is handled
8. how regime scores are calculated
9. how confidence is calculated
10. how model outputs are interpreted

## 4. File 3: `config/sources.yaml`

### 4.1 Source schema

Every source entry should follow this schema:

```yaml
- series_id: STRING
  name: STRING
  provider: FRED
  dimension: STRING
  frequency: daily | weekly | monthly | quarterly
  transform: STRING
  direction: direct | inverse
  weight: FLOAT
  source_tier: 0 | 1
  required: true | false
  stale_after_days: INTEGER
  unusable_after_days: INTEGER
  notes: STRING
```

### 4.2 Direction rule

```yaml
direction: direct
```

means higher transformed values increase the dimension score.

```yaml
direction: inverse
```

means higher transformed values decrease the dimension score.

Example:

- CPI growth is `direct` for `inflation_pressure`.
- VIX is `inverse` for `market_risk_appetite`.
- unemployment rate change is `inverse` for `labor_tightness`.
- credit spread widening is `inverse` for `financial_conditions`.

## 7. Feature Signal Formula

```text
z_score = (transformed_value - rolling_mean) / rolling_std
z_score_clipped = clip(z_score, -3, 3)
bounded_score = tanh(z_score_clipped / 2)
direction_multiplier = +1 if direction == direct else -1
feature_signal = bounded_score * direction_multiplier * feature_weight
```

---

## 8. Dimension Score Formula

```text
dimension_score = sum(feature_signal_i) / sum(abs(feature_weight_i))
```

Clamp final dimension score to `[-1, +1]`.

```text
dimension_score = max(-1, min(1, dimension_score))
```

---

## 9. Dimension Confidence Formula

```text
data_completeness = available_required_weight / total_required_weight
freshness_score = weighted_average(series_freshness_scores)
dimension_confidence = data_completeness * freshness_score
```

### Series freshness score

```text
1.0 = fresh
0.5 = stale but usable
0.0 = unusable or missing
```

---

## 10. Regime Probability Formula

For each regime:

```text
regime_score = sum(dimension_score_j * regime_weight_j)
```

Then:

```text
probability_i = exp(regime_score_i / temperature) / sum(exp(regime_score_k / temperature))
```

The highest probability is the primary regime.

The second-highest probability is the secondary regime.

---

## 11. Transition Zone Logic

```text
transition_zone = true if:
  top_probability < low_conviction_threshold
  OR
  top_probability - second_probability < transition_zone_probability_gap
```

Default values:

```yaml
low_conviction_threshold: 0.35
transition_zone_probability_gap: 0.08
```

## 12. Top Driver Extraction

driver_contribution = dimension_score * regime_weight

Sort by absolute contribution.
